Arrays and Hashes - Organizing Information

October 5th, 2014 by Jorge Rubio C.

Storing, organizing and accessing information is a key feature of computers, knowing that you can input data into a computer, store it in an structured manner and then access it at will has changed information management for good. Data structures are how we organize this information and this week we will cover two types of them, Arrays and Hashes.

Ruby incorporates this two data structures and depending on the task that we face we will prefer using one or the other. Hopefully this post will help us understand why, when and how to use both of them.

Arrays are like lists of elements, they can hold words ("Strings"), numbers("Integers") or even other lists inside them. This gives them great flexibility. Imagine an inventory list, it has many elements in it with one element per slot and you can find what's stored in that slot by searching for that slot's number. Arrays are similar, they are structured like this:

array_1 = (element_1, element_2, element_3, ..., element_n)

And you can access the information inside it by calling the index of the corresponding slot like this:

array_1[1] ---> And its output would be element_2

No, I didn't make a mistake there, that one was on purpose. In Ruby as in many other languages, the first element of a list is the "zeroth" element. In this case the first element of the list would be element_1 and its index would be 0.

If, on the other hand, you wanted to see in which position of the list element_3 is, you would type in something like this:

array_1.at(2) ---> This will output element_3

Hashes on the other side, have a different structure, they can also hold different types of elements, including both arrays and other hashes but the way in which they keep the information in order is different. Imagine a dictionary, its structure allows you to find one element and, connected to it, a second element. Hashes are similar, their structure allows you to list elements in pairs (keys and values), like this:

hash_1 = { element_1 => element_a, element_2 => element_b, ..., element_n => element_x }

Here element_1 would be the key, linked to it, element_a would be the value. Where hashes differ from dictionaries is that you can call its elements in whichever order you prefer. For example, if you want to find out the element associated with element_1, you would do this:

hash_1[element_1] ---> This would output element_a

In the same way, if you wanted to find out the element associated with element_b, you would do this:

hash_1.key(element_b) ---> This would output element_2

From this two main characteristics you can see that both arrays and hashes have different capabilities. Both are flexible in terms of what type of elements they can hold but they differ when it comes to how they handle the information that we store in them.

If you think that you need a list like structure, where you can store elements and be able to access them quickly or figure out in which position they are inside the list, then arrays might be the way to go. If you want to relate elements like in a dictionary and be able to find an element and the element associated to it, then a hash might be the best approach for you.
