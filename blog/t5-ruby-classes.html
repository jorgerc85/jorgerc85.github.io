Ruby Classes - Classic Cars

October 18th, 2014 by Jorge Rubio C.

Have you ever had a big inventory to keep track of? Imagine you owned a car dealership and you wanted to keep track of all your cars, their make, model, year of fabrication and other information. A useful resource to achieve this would be Classes, Ruby Classes to be precise. Throughout this blog post I will try to show you how to achieve this. Ready? Set, GO!

First, let's begin by setting up our Class, a good name for our it would be 'Cars'. Let's define our Cars Class!

class Cars
end

In order to differentiate one car from another you will need to keep record of your cars' make, model and year of fabrication. Every new car we add to our Class will be a new instance of our Class and in order to accept this new car as such we will make it mandatory to input these three pieces of information (make, model and year of fabrication) when creating our new car instance.

class Cars

  def initialize(make, model, year)
  end

end

Our Class is now defined and we can add new instances to it using three pieces of information. To add a new car to our Cars Class we will do it by giving our new car a 'nickname' or short name, for example 'charger' like this:

charger = Cars.new("Dodge", "Charger", 1971)

So far so good, we have our class and we have a way to add new instances to it. But what can we do now with it? As of now, not much but we can change that. Let's add a method that allows us to display the characteristics of a certain car inside our Class.

In order to do this we will have to make our three characteristics into 'instance variables'. Instance variables are variables that are assigned to every instance of a Class at the moment of their creation and that you can access from all methods inside a Class. This is how they are set:

class Cars

  def initialize(make, model, year)
    @make = make
    @model = model
    @year = year
  end

end

Now that our instance variables are created we can access them by using '@variable' from anywhere inside our Class. Let's define our method!

class Cars

  def initialize(make, model, year)
    @make = make
    @model = model
    @year = year
  end

  def display_info
    p "Your car is a #{@make} #{@model} made in #{@year}!"
  end

end

Now that we have this new method inside our Class, let's test it out with our 'charger' instance!

charger.display_info # --> "Your car is a Dodge Charger made in 1971!"

Cool, uh? Now we can add new car instances to our Cars Class and we can display our cars' information whenever we want to. What else could we do? Basically, anything, it all depends on the methods we define inside our Class. To exemplify this I will add two new methods to our Class, the first one will allow us to input a maximum speed for our car and the other one will allow us to calculate the time it would take us to go a certain distance based on that maximum speed. Let's do it!

class Cars

  def initialize(make, model, year)
    @make = make
    @model = model
    @year = year
  end

  def display_info
    p "Your car is a #{@make} #{@model} made in #{@year}!"
  end

  def max_speed(max_speed)     # This method can be condensed even
    @max_speed = max_speed     # further using accessors.
  end                          # Read more about them here and here!

  def time_to_distance(distance)
    time_to_distance = distance / max_speed
    p "Your #{@make} #{@model} will take #{time_to_distance} hours to go     #{distance} miles!"
  end

end

Now that we have that set, let's use those methods. First, we will add our maximum speed:

charger.max_speed(150)

Secondly, we will use our 'time_to_distance' method to do our calculation:

charger.time_to_distance(450) # --> "Your Dodge Charger will take 3 hours to go 450 miles at maximum speed!"

Cool, right? Now you know how to use a Class and what you can do with it. Let's get rolling!
