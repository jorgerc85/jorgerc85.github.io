OOP Concepts - Classes vs Modules

October 26th, 2014 by Jorge Rubio C.

This week I will cover a concept of Object Oriented Programming that I think is very interesting and this is the difference between classes and modules.

Last week I explained classes and how they were useful when trying to model a real life object or system and the different behaviors associated to it.

This week I will explain modules and their use and then I will show you a brief example to compare both of them.

In the hierarchical structure of objects in Ruby, modules are parents to classes, they are a more general type of object and have a more basic function. Classes have the main function of creating instances of itself whenever needed and assign a predefined set of methods to them when they are created.

The main difference that Modules have when compared to Classes is that they can't create instances of itself, they are a collection of methods that you can use to keep them all in one place and assign them to many objects without having to repeat you code everywhere. Let's see an example of this.

class Bird

  def initialize(name)
    @name = name
  end

  def fly
    p "Your #{@name} is flying!"
  end

end

eagle = Bird.new("Eagle")
eagle.fly  # --> "Your Eagle is flying!"

Here you can see a simple class, Bird, and the creation of one instance of it called 'eagle'. Any bird created within the Bird class will be able to fly. This is how classes work, a class with methods in it ensures that all instances created from it can execute those methods. Let's see what a Module looks like.

module AnimalBehavior

  def eat
    p "Your #{name} is eating!"
  end

  def reproduce
    p "Your #{name} has reproduced!"
  end

  def die
    p "Your #{name} just died :("
  end

end

This module contains a set of methods that we call AnimalBehavior, this set of behaviors can be applied to birds but also to other animals so keeping them in one place is a smart strategy to save you time when assigning this behaviors to other classes of animals. Let's see how we can assign this behaviors to Bird

class Bird

include AnimalBehavior

  def initialize(name)
    @name = name
  end

  def fly
    p "Your #{@name} is flying!"
  end

end

eagle = Bird.new("Eagle")
eagle.fly  # --> "Your Eagle is flying!"
eagle.eat  # --> "Your Eagle is eating!"
eagle.die  # --> "Your Eagle just died :("

There are other ways in which you can use Modules to assign behaviors to multiple objects, you can read about them here!

I hope this post has helped you understand a bit better how modules and classes are similar, how they differ and how you can use them together. See you next week!
